Dragon Collection Manager 🐉
Интерактивное консольное приложение для управления коллекцией объектов типа Dragon. Проект выполнен в рамках лабораторной работы №5 по дисциплине "Программирование".

🎯 Особенности
Command Pattern: Каждая команда вынесена в отдельный класс, что обеспечивает чистоту и расширяемость кода.
JSON Storage: Автоматическое сохранение и загрузка коллекции из файла через библиотеку Google Gson.
Interface-based IO: Абстракция над вводом и выводом позволяет легко переключить программу на работу с сетью или GUI.
Strict Validation: Жесткая проверка всех полей (ID, координаты, возраст и т.д.) согласно техническому заданию.
Script Execution: Поддержка выполнения цепочек команд из файлов с защитой от бесконечной рекурсии.
🛠 Технологии
Java 21
Google Gson (библиотека включена в папку lib)
Javadoc для всей публичной части проекта
⌨️ Список команд
help : вывести справку по доступным командам
info : вывести информацию о коллекции
show : вывести все элементы коллекции в строковом представлении
add : добавить новый элемент в коллекцию
update id : обновить значение элемента коллекции по ID
remove_by_id id : удалить элемент из коллекции по ID
clear : очистить коллекцию
save : сохранить коллекцию в файл
execute_script file_name : считать и исполнить скрипт из файла
exit : завершить программу (без сохранения в файл)
remove_last : удалить последний элемент из коллекции
sort : отсортировать коллекцию в естественном порядке
history : вывести последние 10 команд (без аргументов)
remove_all_by_character character : удалить все элементы с заданным характером
print_ascending : вывести элементы в порядке возрастания
print_field_descending_age : вывести значения поля age в порядке убывания
⚙️ Настройка и запуск
Путь к JSON-файлу с данными должен быть передан через переменную окружения MY_LAB_FILE.

Пример запуска (Windows PowerShell):
$env:MY_LAB_FILE="dragons.json" java -jar ru.itmo.nemat.lab5.jar

Пример запуска (Linux / macOS):
export MY_LAB_FILE=dragons.json java -jar ru.itmo.nemat.lab5.jar
