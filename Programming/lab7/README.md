

# Distributed Dragon Collection Manager (Lab 7) 🐉❄️

Распределенное клиент-серверное приложение для управления коллекцией объектов `Dragon` с поддержкой многопоточности, авторизации пользователей и хранения данных в реляционной базе данных (PostgreSQL).

## 🏗 Архитектура проекта
Проект реализован по многомодульной схеме (Maven):
- **common**: Общие классы моделей, сетевые пакеты (Request/Response) и утилиты сериализации.
- **client**: Интерактивное консольное приложение для отправки команд на сервер.
- **server**: Многопоточный сервер, отвечающий за логику команд, работу с БД и управление коллекцией.

## 🎯 Особенности 7-й лабораторной
- **Client-Server TCP**: Взаимодействие через сокеты с использованием кастомной бинарной сериализации.
- **Database Integration**: Хранение коллекции в **PostgreSQL**. Реализована привязка объектов к владельцам.
- **Multi-threading**: 
  - Чтение запросов в отдельных потоках.
  - Обработка команд через `CachedThreadPool`.
  - Отправка ответов через отдельный `FixedThreadPool`.
- **Security**: Система регистрации и логина. Пароли хешируются на стороне сервера (SHA-384/MD2).
- **Concurrent Collection**: Потокобезопасная работа с коллекцией в памяти через `synchronized` блоки и копирование состояний.

## 🛠 Технологии
- **Java 17/21**
- **Maven** (сборка проекта)
- **PostgreSQL** (хранение данных)
- **JDBC** (взаимодействие с БД)
- **java.util.logging** (логирование всех событий на сервере)

## ⌨️ Список команд
### Команды управления:
- `help`, `info`, `show`, `history`, `clear`
- `add`, `update id`, `remove_by_id id`, `remove_last`
- `sort`, `print_ascending`, `print_field_descending_age`, `remove_all_by_character character`
- `execute_script file_name` : выполнение скриптов с поддержкой вложенности.

### Команды сессии:
- `login` : авторизация существующего пользователя.
- `register` : регистрация нового пользователя (с подтверждением пароля).
- `exit` : завершение работы клиента.

## ⚙️ Настройка и запуск

### 1. Подготовка Базы Данных (PostgreSQL)
Необходимо создать таблицы и типы (Enum) согласно схеме. Параметры подключения указываются в `db.properties`.

### 2. Сборка проекта
В корневой папке проекта выполните:
bash
mvn clean install


### 3. Запуск Сервера
Укажите порт через переменную окружения:
bash
export SERVER_PORT=17532
java -jar server/target/server-1.0-SNAPSHOT-jar-with-dependencies.jar

### 4. Запуск Клиента
Передайте адрес и порт сервера как аргументы:
bash
java -jar client/target/client-1.0-SNAPSHOT-jar-with-dependencies.jar localhost 17532

## 🔒 Безопасность
- Доступ к изменению или удалению объектов имеют только их создатели.
- Чтение коллекции (`show`, `info` и др.) доступно всем авторизованным пользователям.
- Скрытый ввод паролей в консоли (через `System.console()`).
