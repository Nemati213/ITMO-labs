# Distributed Dragon Collection Manager - GUI Edition 🐉🖥️🔐

Распределенное клиент-серверное приложение для управления коллекцией объектов `Dragon`. В данной версии полностью переработан клиент: добавлен современный графический интерфейс (GUI), реализована визуализация объектов в реальном времени, а сетевое взаимодействие теперь защищено с помощью SSL-шифрования.

## 🏗 Архитектура проекта
Проект реализован по многомодульной схеме (Maven):
* **common**: Общие классы моделей, сетевые пакеты (Request/Response) и утилиты сериализации.
* **client**: Десктопное графическое приложение (JavaFX) для взаимодействия с сервером.
* **server**: Многопоточный сервер, отвечающий за бизнес-логику, работу с БД и рассылку обновлений клиентам.

## 🎯 Особенности 8-й лабораторной (GUI & Real-time)
* **Графический интерфейс (JavaFX):** Удобное взаимодействие с коллекцией через окна и интерактивные элементы вместо консоли.
* **Интерактивная визуализация (Canvas):** 
  * Драконы отрисовываются на координатной плоскости с учетом их реальных координат и размеров.
  * Цвет объекта уникален и привязан к его создателю (владельцу).
  * Плавные анимации при появлении новых объектов на карте.
  * По клику на объект открывается полная информация о нем с возможностью редактирования.
* **Real-time автообновление:** Любые изменения в коллекции (добавление, удаление, редактирование) мгновенно и автоматически отображаются на экранах всех подключенных клиентов.
* **Умная таблица данных:** Отображение всех полей коллекции. Поддержка мгновенной фильтрации и многоуровневой сортировки колонок под капотом реализована с использованием `Streams API`.
* **Динамическая интернационализация (i18n):** Поддержка нескольких языковых локалей. Переключение языка, а также переформатирование чисел, дат и времени происходит «на лету» без необходимости перезапускать приложение. Данные берутся из `ResourceBundle` (файлы `.properties`).

## 🔒 Безопасность и Сеть
* **SSL/TLS Шифрование:** Весь TCP-трафик между клиентом и сервером надежно зашифрован с использованием SSL-сертификатов (Keystore/Truststore), что защищает данные авторизации и коллекцию от перехвата.
* **Авторизация:** Безопасное окно входа и регистрации. Пароли хешируются на стороне сервера.
* **Разграничение прав:** Редактировать и удалять объекты (как из таблицы, так и с графической карты) может только их создатель.

## 🛠 Технологии
* **Java 17/21**
* **JavaFX** (построение графического интерфейса)
* **Java Secure Socket Extension (JSSE)** (реализация SSL/TLS)
* **Maven** (сборка проекта)
* **PostgreSQL + JDBC** (хранение данных)
* **java.util.logging** (логирование всех событий на сервере)

## ⌨️ Основной функционал клиента
* Окно входа и регистрации с отображением текущего авторизованного пользователя.
* Вкладка **"Таблица"**: просмотр всех объектов, сортировка кликом по заголовку, фильтрация по значениям. Контекстное меню для изменения/удаления.
* Вкладка **"Визуализация"**: графическое представление объектов.
* Окно **создания/редактирования**: удобные формы с валидацией вводимых данных для каждого поля объекта `Dragon`.
* Вызов специфичных команд (удаление по характеристикам, очистка и т.д.) через элементы управления интерфейса.

## ⚙️ Настройка и запуск

### 1. Подготовка Базы Данных и Сертификатов
1. Настройте подключение к PostgreSQL в `db.properties` на стороне сервера.
2. Для работы SSL убедитесь, что сгенерированы файлы ключей (например, `keystore.jks` и `truststore.jks`) и прописаны пути к ним в конфигурации сервера и клиента.

### 2. Сборка проекта
В корневой папке проекта выполните сборку через Maven:
```bash
mvn clean install
```

### 3. Запуск Сервера
Укажите порт и запустите jar-файл сервера:
```bash
export SERVER_PORT=17532
java -jar server/target/server-1.0-SNAPSHOT-jar-with-dependencies.jar
```

### 4. Запуск Клиента (JavaFX)
Запустите собранный клиент (убедитесь, что конфигурация хоста/порта указана верно):
```bash
java -jar client/target/client-1.0-SNAPSHOT-jar-with-dependencies.jar localhost 17532
```
*(Примечание: в зависимости от настроек Maven-плагинов для JavaFX, запуск может осуществляться через `mvn javafx:run`)*
