# Dragon Collection Manager: Client-Server Edition 🐉📡

Сетевое интерактивное приложение для управления коллекцией объектов типа `Dragon`, реализованное с использованием архитектуры **клиент-сервер**. Проект является развитием лабораторной работы №5 и фокусируется на организации сетевого взаимодействия по протоколу TCP.

## 🎯 Особенности
- **Client-Server Architecture**: Четкое разделение ответственности. Клиент обрабатывает пользовательский ввод и отображает результаты, сервер управляет коллекцией и обеспечивает сохранность данных.
- **TCP Networking**: Надежная передача данных между узлами сети с использованием сокетов.
- **Java Object Serialization**: Передача запросов (`Request`) и ответов (`Response`) реализована через стандартный механизм сериализации Java объектов.
- **Modular Structure**: Проект разделен на логические модули:
    - `client`: Командный менеджер клиента и сетевой обработчик.
    - `server`: Бизнес-логика, менеджер коллекции и обработчик соединений.
    - `common`: Общие классы моделей, исключения и сетевые пакеты.
- **Server Logging**: Все события на сервере (подключения, выполнение команд, критические ошибки) логируются с использованием `java.util.logging`.
- **JSON Persistence**: Сохранение коллекции на стороне сервера в формате JSON через библиотеку Google Gson.

## 🛠 Технологии
- Java 17 (совместимость с Helios)
- Java Object Serialization (сетевой протокол)
- Google Gson (только для файлового хранения на сервере)

## ⌨️ Список команд
- **Локальные команды клиента**: `help`, `history`, `execute_script`, `exit`.
- **Команды, исполняемые на сервере**: `info`, `show`, `add`, `update`, `remove_by_id`, `clear`, `remove_last`, `sort`, `remove_all_by_character`, `print_ascending`, `print_field_descending_age`.

## ⚙️ Настройка и запуск

### Сервер (Server)
Для работы сервера необходимо задать переменные окружения:
- `SERVER_PORT`: Порт для прослушивания соединений.
- `MY_LAB_FILE`: Путь к файлу коллекции (JSON).

**Запуск (PowerShell):**
`$env:SERVER_PORT=12345; $env:MY_LAB_FILE="data.json"; java -jar server-jar.jar`

**Запуск (Linux/Helios):**
`SERVER_PORT=28435 MY_LAB_FILE=data.json java -jar server-jar.jar`

### Клиент (Client)
Клиент принимает параметры подключения через аргументы командной строки:
1. Хост сервера (например, `localhost` или `helios.cs.ifmo.ru`).
2. Порт сервера.

**Запуск:**
`java -jar client-jar.jar localhost 12345`
